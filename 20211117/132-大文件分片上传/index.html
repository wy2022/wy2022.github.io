132-大文件分片上传

发表于 2021-11-17 分类于 js 阅读次数：
大文件分片上传

分片上传

为了方便大家阅读和理解，我将以单个大文件上传为例，先简单描述下思路。

antd 的上传组件有一个上传前的钩子，里面是可以拿到 file 信息，上传前将 file 切片，然后包装成一个一个的请求，放到一个数组，上传的的时候将数组的请求执行就可以了，执行完后发送一个合并请求，我没有用 Promise.all 去执行，而是 2 个 2 个的递归执行。

对大文件先通过 slice 进行切片

核心是利用 Blob.prototype.slice 方法

createFileChunk接收两个参数
dataSource:所上传的 File 大文件，size：每个分片大小

hash由文件名和序号组成，后端合并的时候需要按顺序合并。

this.createHttp方法分析
简单的做了参数处理，this.request 里面才是真是 ajax 请求
onProgress：监听 ajax 进度并实时记录下来

为每个分片创建一个 http 请求

this.request方法通过 promise 和 ajax 包装
url:分片上传接口。data：分片参数。onProgress：监听此分片上传进度。
requestList：所有正在上传的分片请求集合。（断点续传用的）
也可以在此方法里面设置 token 认证

记录分片进度方法

this.createProgressHandler(data, hash)在上面 createHttp 方法调用

调用开始上传的方法

this.upFile(this,state.fileChunkList[0])(true)
参数 true 是保证 2 个请求一发

合并分片的方法我就不写了，就调用一个接口即可。

假如存在上传失败的分片，会被记录在 fileChunkList[0].errChunkFile.对这个失败的数组做一个上传就可以了。

断点续传 暂停

this.state.requestList 是当前正在请求的分片集合。暂停就是把请求 abor

续传，可以获取已经上传成功的，然后把未上传的重新上传即可。

我只写了前端的大致实现思想，后端只需提供单个分片上传的接口，合并分片的接口。我的 hash 用文件名+索引，用 spark-md5 对文件内容生成一个 hash 才是最合适的。

单个大文件上传感觉其实并不复杂，知道它的大致思想再去扩展多文件排队上传，断点续传，记录每个文件的进度条、总进度条甚至每个分片的进度条，还要考虑暂停的时候，由于 onProgress 是实时监听进度条的，当分片上传了百分之 80，取消后变为 0，进度条回退的情况….

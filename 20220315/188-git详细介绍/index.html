Git 介绍

免费开源的分布式版本控制工具

版本控制：记录文件内容变化，以便将来查阅特定版本修订情况的系统(记录文件修改历史记录，从而让用户能够查看历史版本，方便版本切换)

三个区：

  • 工作区：代码存放的磁盘的目录的位置（写代码）

  • 暂存区：增删没有记录（临时存储）

  • 本地库：代码无法再删除（历史版本）

    • 工作区-> git add -> 暂存区-> git commit -> 本地库 -> push -> 远程库

  • 远程库：从本地将代码推送到远程库

代码托管中心：基于网络服务器的远程代码仓库，简称为远程库

Git 安装

官网：https://git-scm.com/

安装时全程默认即可

安装目录最好是非中文，目录名不带空格

Git 命令

常用命令
命令名称作用
git config –global user.name 用户名设置用户签名(真实与否皆可，首次安装必须设置，否则无法提交代码)
git config –global user.email 邮箱设置用户签名(和将来登陆的 github 或其它代码托管中心的账号没有任何关系，只是用来区分不同操作者的身份)
git init初始化本地库
git status查看本地库状态
git add 文件名添加到暂存区
git commit -m “日志信息” 文件名提交到本地库
git reflog查看历史记录
git reset –hard 版本号版本穿梭

Git 分支

在版本控制过程中，同时推进多个任务，为每个任务，创建每个任务单独的分支

优点：

  • 同时并行推进多个功能开发，提高开发效率
  • 各个分支开发过程中互不影响
分支操作
命令名称作用
git branch 分支名创建分支
git branch -v查看分支
git checkout 分支名切换分支
git merge 分支名把指定的分支合并到当前分支上
git push 远程地址别名(或远程地址) 分支名将新建的分支同步到远程仓库

合并分支(冲突合并)

两个分支进行合并，但它们含有对==同一个文件的同一个位置==有完全不同的修改，则在合并时出现冲突，git 无法决断该保留该文件哪个分支的修改。
这种情况下 git 会保留两个文件的内容，由自己手动修改文件，决定留哪个内容后，再重新提交

Github

官网：https://github.com/

远程仓库操作常用命令
命令名称作用
git remote -v查看当前所有远程地址别名
git remote add 别名 远程地址为远程地址取别名(连接到远程地址)
git remote remove 远程地址别名删除别名
git push -u 别名 分支推送本地分支上的内容到远程仓库(没有别名时使用链接)
git clone 远程地址将远程仓库的内容克隆到本地
git pull 远程仓库地址别名 远程分支名将远程仓库对于分支最新内容拉下来后与当前本地分支直接合并
git pull 远程仓库别名 远程分支名 –allow-unrelated-histories如果直接 pull 不行可在后面加上–allow……命令
创建远程库

  1. 打开官网

  2. 右上角登陆 sign in

  3. 点击右上角+号，New repository,创建远程仓库

  4. 输入远程仓库名字(一般与本地仓库一致),其它默认，点击下方 Create repository,创建仓库

  5. 创建成功后可看到两个链接(HTTPS、SSH),复制远程库链接(此处先复制 https)

  6. 为远程仓库创建别名(别名最好与远程仓库的名字保持一致)

  7. git remote -v 查看别名

代码克隆 clone

  1. 新建文件夹(拉取的远程仓库代码将会放到这里)

  2. 进入新建的文件夹，打开 git 命令窗口

克隆下来的代码修改后，执行 push 命令，需要先登陆账号，并且账号在团队中，才能成功 push

执行克隆命令后会自动做三件事情

  • 拉取代码
  • 初始化本地仓库(会在当前文件夹下新建一个与远程仓库名字相同的文件夹，里面放的代码)
  • 创建远程地址的别名(默认为 origin)
团队内协作

邀请其它成员加入团队：

  • 进入远程仓库
  • 点击 settings
  • 点击 Manage access
  • 点击绿色按钮，Invite collaborator
  • 在弹出的窗口中输入要邀请的成员的 github 账号
  • 添加账号到此仓库
  • 点击 Pending Invite，复制邀请函(一个链接地址)

被邀请的成员

  • 登陆进 github
  • 将邀请函地址粘贴到地址栏，确定
  • 点击绿色按钮 Accept invitation 同意邀请,(Decline:婉拒)
  • 此时就可以看到邀请人的仓库
  • 已具有代码推送权限，可进行代码推送
跨团队协作

其它团队被邀请人

  • 登陆进 github
  • 将邀请函地址粘贴到地址栏，确定
  • 点击右上角 Fork
  • 此时仓库下面就有了邀请人的远程仓库

修改代码后推送到邀请人仓库：

  • 进入仓库，点击 Pull requests
  • 点击右边绿色按钮,New pull request
  • 点击右边绿色按钮,Create pull request
  • 写提交信息
  • 留言
  • 点击右下角绿色按钮,Create pull request
  • 邀请人账号登陆进入仓库时，会看到 Pull requests 有个 1,点击
  • 点击查看
  • 点击 Merge pull request,同意推送请求,点击 Confirm merge,确认同意合并

微信小程序

准备

官网：https://developers.weixin.qq.com/miniprogram/dev/framework/

小程序大小不能超过 8M

小程序开发主要分成三部分：

  • 页面布局：WXML
  • 页面样式：WXSS
  • 页面脚本：JavaScript + WXS

开发过程：

第一个小程序应用

小程序：一个大的 App，App 下面有多个 page，每个 page 多个组件

小程序工具栏普通编译->添加编译模式，可选择指定的页面作为启动页面

App -> 多个界面 -> 多个组件

文件结构：

  • App
    • app.js //创建 App 实例的代码以及一些全局相关的内容
    • app.json //全局的一些配置，比如 window/tabbar 等
    • app.wxss //全局的一些样式配置
  • Page
    • page.js //创建 page 实例的代码，以及页面相关的内容
    • page.json //业务单独的配置，比如页面对应的 window 配置,usingComponents
    • page.wxml //页面的 wxml 布局代码
    • page.wxss //页面的样式配置
  • Component
    • component.js //创建 component 实例的代码，以及组件内部的内容
    • component.json //组件内部的配置，比如当前组件使用了别的组件
    • component.wxml //组件的 wxml 布局代码
    • component.wxss //组件的样式配置

小程序不允许 DOM 操作

数据绑定：使用 Mustache 语法，双大括号

列表渲染：wx:for 循环

单标签需以 /> 结尾

M:Model V:View VM:ViewModel

project 和 sitemap

app.json

pages：页面路径列表

  • 用于指定小程序由哪些页面组成，每一项都对应一个页面的路径信息
  • 小程序中的都必须在 pages 中进行注册
  • 手动写注册路径，保存之后会自动新建一个 pages

window：全局默认的窗口展示

  • 用户指定窗口如何展示，其中包含很多其它属性

tabBar：顶部 tab 栏的展示

编译模式：指定编译的启动页，不用每次都在首页

小程序双线程模型

渲染层：执行 wxml 和 wxss 文件，使用 WebView 线程渲染，一个页面使用用一个 WebView

逻辑层：执行 js 文件，使用 jsCore 运行 JS 脚本

WXSS&WXML&WXS

样式的三种写法

行内样式、页面样式、全局样式

app.wxss 里面的都为全局样式

三种样式作用域同一个组件优先级：行内样式 > 页面样式 > 全局样式

选择器权重：!important > style=”” > #id > .class > element

0.5px = 1rpx (iPhone6 上)

建议：开发微信小程序时使用==iPhone6==作为视觉稿的标准

WXML

Mustache 语法

可以写成单标签，也可以写成双标签

必须有严格的闭合：没有闭合会导致编译错误

大小写敏感:Class 与 class 不同

逻辑判断

wx:if 隐藏时，组件根本不会创建

hidden 属性

hidden 将组件隐藏起来时，组件依然存在，hidden 属性为 false 时组件会显示出来

显示和隐藏切换的频率非常高，选择使用 hidden，切换频率低就选择 wx:if

列表渲染-wx:for

遍历数组/字符串/数字

block 标签

不是单个组件，仅仅是一个包装元素，不会在页面中做任何渲染，只接收控制属性

某些情况下，使用 wx:if 或者 wx:for 时，需要包 裹一组组件标签

希望对这一组标签进行整体的操作，此时可使用 block 标签包裹

作用类似于 Vue 的 template，react 的 fragement<></>

模板

template，可以在模板中定义代码片段，在不同的地方调用

之前小程序不支持自定义组件时，为了代码地复用而使用

使用==name 属性==，作为模板的名字，然后在 template 内定义代码片段

模板包裹的内容，在没有使用时是不会进行任何的渲染的

WXS

https://developers.weixin.qq.com/miniprogram/dev/framework/view/wxs/

WeiXin Script,小程序的一套脚本语言，结合 wxml，可以构建出页面的结构

  • 在 WXML 中不能直接调用 Page/Component 中定义的函数
  • WXS 中不能调用其它 JavaScript 文件中定义的函数，也不能调用小程序提供的 API
  • WXS 函数不能作为组件事件的回调
  • WXS 在 IOS 比 js 快 2~20 倍，安卓无差别

属性类型说明
typeString事件类型
timeStampInteger页面打开到触发事件所经过的毫秒数
targetObject触发事件的组件的一些属性值集合
currentTargetObject当前组件的一些属性值集合
detailObject额外的信息(鼠标点击时距离左边和上边的距离)
touchesArray触摸事件，当前停留在屏幕中的触摸点信息的数组(记录触摸的手指数量)
changedTouchesArray触摸事件，当前变化的触摸点信息的数组(记录触摸变化的手指数量)

touches 和 changedTouches 的区别

touches：用来记录当前有几个手指在小程序中触摸以及对应的触摸点的信息

changedTouches：记录变化的(原来有一个，现在两个手指在触摸，数组就为 1)

  1. 在 touchend 中不同
  2. 多手指触摸中不同

currentTarget 和 target 的区别

currentTarget:记录触发事件的 view

target：记录产生事件的 view

组件化

自定义组件：https://developers.weixin.qq.com/miniprogram/dev/framework/custom-component/wxml-wxss.html

  • 因为 wxml 标签名只能是小写字母、中划线和下划线的组合，所以自定义组件的标签名只能包含这些字符
  • 自定义组件也可以引用自定义组件
  • 自定义组件和页面所在项目根目录名不能以 wx-为前缀
  • 在 app.json 注册的组件就是全局组件，可在任意界面使用
样式影响
  • 组件内或者外的 class 样式默认局部生效，不会影响到其它
  • 外部使用标签选择器会对组件内产生影响，其它不会
  • 组件内不能使用 id 选择器，属性选择器，标签选择器

页面跳转

小程序界面跳转有两种方式:

  • 通过 navigator 组件

    • redirect：关闭当前页面，跳转到应用内某个页面。但是不能跳转到 tabbar 页面，并且不能返回（不是一个压栈）
    • switchTab：跳转到 tabBar 页面，并关闭其他所有非 tabBar。（需要在 tabBar 中定义的）
    • reLaunch：关闭所有页面，打开应用中某个页面。（直接展示某个页面，并且可以跳转到 tabBar 页面）

  • 通过 wx 的 API 跳转

类型别名和接口之间的差异

类型别名和接口非常相似，在很多情况下你可以自由选择它们。几乎所有的功能都在interface中可用type，关键区别在于不能重新打开类型以添加​​新属性与始终可扩展的接口。

type 和interface区别

interface 扩展接口

type 扩展接口

interface添加新字段

type 创建后不可以修改

泛型

泛型看这

never用处

文档出处

返回never的函数必须存在无法达到的终点

举例：

  interface Foo {
    type: 'foo'
  }

  interface Bar {
    type: 'bar'
  }

  type All = Foo | Bar

在 switch 当中判断 type，TS 是可以收窄类型的 (discriminated union)：

  function handleValue(val: All) {
    switch (val.type) {
      case 'foo':
        // 这里 val 被收窄为 Foo
        break
      case 'bar':
        // val 在这里是 Bar
        break
      default:
        // val 在这里是 never
        const exhaustiveCheck: never = val
        break
    }
  }

注意在 default 里面我们把被收窄为 never 的 val 赋值给一个显式声明为 never 的变量。如果一切逻辑正确，那么这里应该能够编译通过。但是假如后来有一天你的同事改了 All 的类型：

type All = Foo | Bar | Baz

然而他忘记了在 handleValue 里面加上针对 Baz 的处理逻辑，这个时候在 default branch 里面 val 会被收窄为 Baz，导致无法赋值给 never，产生一个编译错误。所以通过这个办法，你可以确保 handleValue 总是穷尽 (exhaust) 了所有 All 的可能类型。

  function markDate(timestrap: number): Date;

  function markDate(m: number, d: number, y: number): Date;
  function markDate(mOrTimestrap: number, d?: number, y?: number): Date {
    if (d !== undefined && y !== undefined) {
      return new Date(y, mOrTimestrap, d)
    } else {
      return new Date(mOrTimestrap)

readonly

使用readonly修饰符并不一定意味着一个值是完全不可变的——或者换句话说，它的内部内容不能改变。这只是意味着属性本身不能被重写。

通俗的讲，对象的属性可以修改，但是不能直接重新赋值整个对象

  interface Home {
    readonly resident: { name: string, age: number }
  }

  function changeHome(home: Home) {
    home.resident.name = '123'
  }
  //下面是不可以的
  function changeHome1(home: Home) {
    home.resident = {
      name: 'ddd',
      age: '222'
    }
  }

接口继承 interface extends

某些情况下，接口有才重复的类型，可以利用继承少些点属性，可以继承多个

  interface BasicAddress {
    name: string,
    city: string,
    phone: number
  }

  interface newAddress extends BasicAddress {
    unit: string
  }


  const add = (address: newAddress) => {
    console.log(address);

  }


  add({ name: 'sss', city: 'zhongguo', phone: 111, unit: 'sss' })

enum 枚举

使用枚举，可以定义一些带名字的常量。用于清晰地表达意图或创建一组有区别的用例。

  //数字类型枚举：默认情况下，第一个枚举值是0，后续至依次增1
  enum Direction {
    LEFT,
    RIGHT,
    TOP,
    BOTTOM
  }

  function controlDirecton(direction: Direction) {
    switch (direction) {
      case Direction.LEFT:
        console.log("left");

        break;
      case Direction.BOTTOM:
        console.log("bottom");
        break;
      case Direction.RIGHT:
        console.log("right");
        break;
      case Direction.TOP:
        console.log("top");
        break;
      default:
        let direc: never = direction
        break;
    }
  }

  controlDirecton(Direction.TOP)
